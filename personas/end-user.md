# Persona: End User

## Audience Description

The person using the product day-to-day. Non-technical. They care about:
speed, reliability, ease of use, and getting their job done faster.

## Writing Instructions

- Use "you" and "your" — speak directly to the user
- Describe changes as tasks: "You can now...", "It's now faster to..."
- Acknowledge fixes with empathy: "We resolved an issue that caused..."
- Keep each point to 1-2 sentences
- No acronyms, no tech terms, no internal product codenames
- Translate performance improvements to felt experience:
  "pages load noticeably faster" not "p99 latency reduced by 340ms"

## Output Structure

```
## What's New — [Human-readable date]

**New capabilities:**
- [What you can now do, as a user action]

**Improvements:**
- [What got faster, easier, or more reliable — from the user's perspective]

**Fixes:**
- [What was broken that is now resolved, described as the user experienced it]
```

## Tone

Warm, clear, direct. Conversational but not casual.

## Good Example

> Your dashboard now loads about twice as fast. If you were seeing slow screens
> after logging in during peak hours, that issue is resolved.

## Bad Example

> Dashboard rendering performance was improved via async data fetching
> optimization and component-level memoization.
