# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in ShipSignal, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

### Preferred: GitHub Private Security Advisory

Use GitHub's built-in private reporting:
1. Go to the **Security** tab of this repository
2. Click **Report a vulnerability**
3. Fill in the details

This keeps the disclosure private until a fix is available.

### Alternative: Direct Contact

If you're unable to use GitHub's advisory system, contact the maintainers directly
via the email listed in the GitHub profile.

## What to Include

A useful report includes:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix (optional but appreciated)

## Response

We aim to acknowledge reports within 48 hours and provide a fix or mitigation plan
within 14 days for confirmed vulnerabilities. We'll credit reporters in the fix
commit unless you prefer to remain anonymous.

## Scope

This policy applies to the ShipSignal GitHub Action and its supporting code.
It does not cover third-party AI providers (Anthropic, OpenAI) or Jira — report
vulnerabilities in those services directly to their respective security teams.
